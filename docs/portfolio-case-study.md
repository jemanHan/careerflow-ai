# CareerFlow AI — 포트폴리오용 개발 기록 (과정·의사결정·문제 해결)

이 문서는 **면접·포트폴리오 설명용**으로, “무엇을 없앴고·무엇을 바꿨고·왜 그랬는지·막혔을 때 어떻게 풀었는지”를 한곳에 모은 요약입니다. 세부 타임라인은 `docs/dev-log.md`, 기술·제품 하이라이트는 `docs/portfolio-points.md`를 함께 보시면 됩니다.

---

## 1. 한 줄 요약

**채용 지원 준비**를 단계형 제품으로 풀었다: 입력 → 공고 대비 장·단점 스냅샷 → 대화형 보완 → 문서 초안 → 면접 대비 카드.  
백엔드는 **NestJS + Prisma + PostgreSQL**, AI는 **LangChain 단계별 체인 + 단계별 모델 라우팅**, 프론트는 **Next.js 최소 UI**로 E2E 검증 가능한 MVP다.

---

## 2. 구조를 단순화한 일 — “지운 것”

| 제거·통합 대상 | 이유 | 결과 |
|----------------|------|------|
| **`llm` 모듈** (초기 중복 경로) | LLM 호출 경로가 `langchain` 워크플로와 이중화되면 유지보수·장애 추적이 어려움 | **`LangchainWorkflowService` 단일 오케스트레이션**으로 통합 |
| **`applications` 모듈** | 소스·분석 흐름과 역할이 겹침 | **`source-documents` + 각 도메인 서비스**로 정리 |
| 결과 화면 **Raw JSON / 중복 네비** 등 | 데모·신뢰도에는 도움이 적고 노이즈 | **서버 `WorkflowRun` 로그에 일원화**, UI는 카드·핵심 메타데이터 위주 |
| 장·단점 영역의 **휴리스틱 “적합도 점수·델타”** | 근거 없는 숫자는 오히려 신뢰를 깎음 | **`fitAnalysisJson`은 서술형 스냅샷**(강점·약점·보완 포인트·고지)만 유지 |

“삭제”는 기능 축소가 아니라 **단일 진실 원천(single orchestration, single log story)** 을 만드는 리팩터링에 가깝다.

---

## 3. 모델·프로바이더 쪽 변화 — “바꾼 것”

- **Provider**: 로컬/데모 기본을 **Gemini Developer API**(`LLM_PROVIDER=gemini`)로 두고, OpenAI는 선택 사항으로 유지.
- **이중 슬롯(비용·품질 균형)**  
  - **Light (`GEMINI_DEFAULT_MODEL`)**: 후보 추출, JD, 갭, 후속 질문, 면접 질문 등 **고빈도·구조화 위주** 단계.  
  - **Quality (`GEMINI_HIGH_QUALITY_MODEL`)**: **문서 생성·채용공고 맞춤 리라이트** 등 **긴 생성·톤이 중요한** 단계.  
  - (설정에 따라) **최초 공고 대비 분석 1회**에만 쓰는 **Premium** 슬롯(`GEMINI_PREMIUM_MODEL`)을 두어, “첫인상 분석”만 품질을 올리는 선택지를 둠 — 미설정 시 해당 경로는 비활성.
- **면접 질문 단계의 tier**: 한동안 quality로 올렸다가, **비용·호출 패턴**과 제품 역할(서류 보완 vs 면접 심층)을 재정의하면서 **light 쪽에 두는 설계**로 재정렬한 이력이 있음( `docs/dev-log.md` 2026-03-24 항목 참고).
- **장애 시**: 외부 API **429·quota** 등은 `WorkflowRun`에 **fallback 여부·사유**를 남기고, 가능하면 워크플로가 끊기지 않게 **graceful fallback** 경로를 유지.

---

## 4. 제품·데이터 모델 쪽 눈에 띄는 변화

- **`fitAnalysisJson`**: 갭 분석 결과를 바탕으로 **강점·약점·개선 포인트·고지 문구**를 저장. 수치 점수 대신 **설명 가능한 서술**에 집중.
- **`WorkflowRun`**: 단계별 입력 요약·**provider/model 라우트**·fallback 메타데이터를 남겨 “왜 이 결과가 나왔는지”를 **DB로 추적** 가능하게 함.
- **라이트 테스트 계정(숫자 3자리 등)**: 비로그인 데모에서 **저장·재방문**이 되도록 `Application`과 연결(프로덕션 인증 대체가 아님을 문서에 명시).

---

## 5. 프론트엔드 UX (최근)

- **전역 네비**(`GlobalAppNav`): 뒤로가기·홈·새 워크플로·내 워크플로로 이동 일관화.
- **입력 초안**: `/new`의 네 개 주요 텍스트(및 테스트 ID)는 **`localStorage`에 디바운스 저장**, 재방문·뒤로가기 후에도 복구. 제출 성공 시 초안 삭제.
- **대화형 보완 답변**: 지원서(`applicationId`) 단위로 **`sessionStorage` 초안** + 첫 빈 저장으로 덮어쓰기 방지(스킵 가드).

## 6. 문제 해결 노트 (대표 사례)

문제 → 원인 가설 → 조치 → 결과 형식으로 정리했다. 상세는 `docs/troubleshooting.md`에 더 많다.

| 문제 | 원인 | 조치 | 결과 |
|------|------|------|------|
| `npm install` 의존성 트리 충돌 | `langchain` / `@langchain/*` 버전 충돌 | 불필요한 직접 의존성 정리 | 설치·빌드 안정화 |
| Prisma `P1000` | `DATABASE_URL`과 로컬 DB 계정 불일치 | `.env` 정합성 맞춤 | 마이그레이션·연결 성공 |
| Nest **DI 실패** (예: `RequestRateLimiterService`) | 공통 provider가 모듈 밖에 안 보임 | `@Global()` `CommonModule`로 제공·export | 서버 기동 정상 |
| **OpenAI 429 / quota** | 빌링·한도 | 예외 시 fallback으로 단계 지속 + 로그 | 데모 플로우 중단 완화 |
| **Gemini 429** | 무료 티어 한도·재시도 대기 | 재시도 정책 조정(예: 불필요한 대기 축소), 문서에 한도 설명 | 체감 지연·반복 실패 완화 |
| **갭/장단점 신호가 비어 보임** | UI-safe 필터가 단일 기술 토큰까지 과도 제거 | 단일 기술 신호 allowlist·백업 추론 등 (`fit-analysis`·`signal-display` 계열) | 빈 배열 붕괴 완화 |
| **후속 답변 초안이 빈 객체로 덮임** | hydration 직후 persist 타이밍 | “첫 저장 한 번 스킵” ref 가드 | 초안 유실 방지 |

## 7. 면접에서 말하기 좋은 문장 (예시)

- “LLM을 한 번에 때리는 게 아니라 **단계별 체인**으로 나누고, **모델 tier**를 나눠 비용과 품질의 균형을 잡았다.”
- “중복 모듈을 없애 **단일 워크플로 서비스**로 모은 이유는 장애 추적과 변경 비용을 줄이기 위해서다.”
- “수치 점수 대신 **설명 가능한 스냅샷**을 저장해, 근거 없는 숫자로 신뢰를 깎지 않게 했다.”
- “429·quota는 **fallback + 메타데이터 로깅**으로 운영 가능한 수준까지 끌어올렸다.”
