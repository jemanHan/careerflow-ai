# Portfolio Points

## 제품 관점
- 채용 지원 워크플로우를 실제 사용자 문제(문서 품질/정합성) 중심으로 구조화
- 분석 결과를 후속 질문으로 연결해 반복 개선 루프 구현

## 기술 관점
- 단일 프롬프트가 아닌 LangChain 모듈형 체인 설계
- NestJS + Prisma + PostgreSQL로 E2E 저장/조회 흐름 구현
- `WorkflowRun`으로 단계별 AI 실행 증거 로그 저장
- Next.js 최소 UI에서 입력/분석/후속질문/문서/면접질문 플로우 직접 검증 가능
- 실행 중 장애(`npm` 미인식) 원인/대응 문서화
- OpenAI 단일 공급자 전략으로 MVP 복잡도 제어
- 단계별 모델 라우팅(기본/고품질)으로 비용-품질 균형 최적화
- 레이트 리미팅/동시실행 락/중복 재실행 스킵으로 과도한 API 사용 방지

## 인터뷰 어필 포인트
- “왜 이 구조가 실무에서 운영 가능한가”를 설명 가능한 코드/문서로 제시
- 중복 아키텍처(`llm` vs `langchain`)를 제거하고 단일 오케스트레이션으로 정리한 리팩터링 근거
- "모델 라우팅 + 과호출 방지"를 통해 AI 제품의 비용/안정성 문제를 실무적으로 다룬 경험
