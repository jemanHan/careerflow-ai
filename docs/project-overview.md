# CareerFlow AI — 프로젝트 개요

## 문제 정의

취업 준비에서 시간이 많이 드는 구간은 **지원자 정보와 채용공고를 맞춰 문서를 반복 수정**하는 작업입니다.  
CareerFlow AI는 이를 단일 생성이 아닌 **단계형 워크플로우**로 구성하고, 중간 판단 근거와 실행 로그를 함께 남기는 것을 목표로 합니다.

## 제품 목표

- 이력서·포트폴리오·JD 입력을 기반으로 **분석 → 보완 → 문서 초안 → 면접 준비**를 한 흐름으로 제공  
- 결과뿐 아니라 중간 근거(`WorkflowRun`, `fitAnalysisJson`)를 저장해 **설명 가능성** 확보  
- 데모 환경에서도 테스트 ID 기반 **저장·재방문** UX로 제품성 검증  

## 핵심 가치

- **분해 가능한 AI 실행**: one-shot 대신 체인 기반 단계 실행  
- **근거 중심 개선 루프**: 갭 탐지 → 후속 질문 → 재분석  
- **운영 가능한 구조**: 모델 라우팅, 과호출 방지(레이트리밋·락), 실패 시 로그·복구 전략  

## MVP 범위

- 소스 입력·저장, 테스트 ID(3자리) 연동  
- 후보자·채용공고 구조화, 갭 분석, 공고 대비 스냅샷(`fitAnalysisJson`)  
- 대화형 후속 보완(질문·답변)  
- **문서 초안 2종**: 지원동기/자기소개 초안(`coverLetter`), 경력기술서 초안(`careerDescription`)  
- 선택 단계: 채용공고 맞춤 리라이트(`rewriteForJob`, API 지원 — UI는 설정에 따라 생략 가능)  
- 면접 대비 리포트(핵심 3 + 심화 2)  
- 테스트 계정 기반 저장·재조회(비인증 데모)  

## 산출물 원칙

- 문서와 면접 리포트는 사용자가 다시 수정하고 다듬을 수 있는 작업 초안으로 제공  
- 결과보다 반복 보완과 재활용이 가능한 워크플로우 구조를 우선
