# 1일차 개발일지 (2026-03-23 18:00 ~ 22:00)

## 구현 범위
- 백엔드
  - LangChain 단계형 워크플로우(분석/후속질문/문서생성/면접질문/리라이트) 실행 안정화
  - `WorkflowRun`에 단계별 실행 메타데이터(`llmRoute`, `llmExecution`) 기록
  - Gemini 모델 라우팅(light/quality) 검증
  - candidate 체인 파싱 정규화(`projects[].evidence` string -> string[])
  - 후속 질문 개수 5개 고정
- 프론트엔드
  - 결과 페이지 카드형 레이아웃 정리
  - "현재 상태 요약" 섹션(경력 레벨/특화 포인트/보완점)
  - "왜 이런 결과가 나왔나" 근거 요약 추가
  - 후속 질문 1:1 답변 입력 폼으로 변경
  - 강조 프로젝트 영역 접기/펼치기 처리
  - 로딩 상태 스피너/진행 메시지 추가
  - API 실행 로그/Raw JSON은 화면에서 제거

## 주요 트러블 포인트와 해결
- 포트 충돌 및 중복 서버 실행
  - 증상: `EADDRINUSE`로 백엔드 재기동 실패, 중복 터미널 다수 발생
  - 해결: 포트 확인 -> 기존 PID 종료 -> 단일 인스턴스 재기동 원칙 적용
- Gemini 라우팅 표시 대비 fallback 고정
  - 증상: `llmRoute`는 Gemini인데 결과는 fallback 텍스트
  - 원인: 런타임에서 provider key 인식 불일치
  - 해결: `llmExecution` 진단 메타데이터 추가, 부팅 로그로 key 인식 확인
- candidate 단계만 fallback
  - 증상: 다른 단계 일부 성공 시에도 후보자 추출은 fallback
  - 원인: LLM 출력의 `projects[].evidence` 타입 불일치
  - 해결: 유연 스키마 파싱 후 배열 정규화
- 결과가 나와도 사용자 입장에서 해석 어려움
  - 증상: 생성 결과/근거/상태 확인이 어려움
  - 해결: 결과 페이지 정보 구조 재배치 + 근거 요약 UI 도입
